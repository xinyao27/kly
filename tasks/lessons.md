# Lessons

## 2026-03-18

- When discussing roadmap priority, do not default to release or publish work just because the package is technically close to publishable.
- First check the user's product intent: if the user signals that details, UX, or implementation quality are not ready, prioritize polish and quality closure ahead of shipping tasks.
- Treat publish readiness as a later-stage multiplier, not the default next step.
