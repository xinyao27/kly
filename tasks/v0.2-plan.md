# kly v0.2 执行计划

> 目标：kly 从独立 CLI 工具升级为可观测性系统核心组件
> 两大方向：(1) 新增核心库函数 (2) CLI 全面改造为 agent-first

---

## 总览

```
Phase 1 — 核心库函数        ← 新增能力，Santi 直接 import 用
Phase 2 — CLI 全面改造       ← agent-first 输出，去 MCP，去 clack
Phase 3 — 清理 & 测试        ← 删除废弃代码，补测试
```

## Phase 1：核心库函数

### 1.1 `dependencies` 表 + 构建时写入

**改动文件：**

- `src/database.ts` — schema 新增 `dependencies` 表 + CRUD 方法
- `src/indexer.ts` — build 完成后同步写入依赖关系

**数据库变更：**

```sql
CREATE TABLE IF NOT EXISTS dependencies (
  from_path TEXT NOT NULL,
  to_path TEXT NOT NULL,
  PRIMARY KEY (from_path, to_path)
);

CREATE INDEX IF NOT EXISTS idx_deps_to ON dependencies(to_path);
```

**IndexDatabase 新增方法：**

```typescript
// 写入
upsertDependencies(fromPath: string, toPaths: string[]): void
// 删除某文件的所有出边
removeDependencies(fromPath: string): void
// 正向查询：这个文件 import 了谁（resolved paths）
getDependencies(filePath: string): string[]
// 反向查询：谁 import 了这个文件
getDependents(filePath: string): string[]
```

**indexer.ts 变更：**

在 `indexFiles()` 函数末尾（upsertFiles 之后），解析每个文件的 imports，resolve 为实际路径，写入 dependencies 表。复用 `graph.ts` 的 `resolveImport()` 逻辑——需要把 `resolveImport` 从 graph.ts 的私有函数提升为导出函数。

在 `buildFromGitDiff()` 中删除文件时，同步删除该文件的依赖关系。

在 `buildClassic()` 末尾清理不存在文件时，同步清理 dependencies 表。

**注意：** 旧的 .kly/db/\*.db 没有 dependencies 表。`init()` 里用 `CREATE TABLE IF NOT EXISTS` 就行，下次 build 会自动填充。但旧数据不会自动有依赖关系，需要 `kly build --full` 重建。

### 1.3 `getFileHistory()` 函数

**改动文件：**

- `src/git.ts` — 新增 `getFileHistory()`

**新增类型（`src/types.ts`）：**

```typescript
interface GitCommit {
  hash: string;
  author: string;
  email: string;
  date: number; // unix timestamp
  message: string;
}
```

**实现：**

```typescript
export function getFileHistory(root: string, filePath: string, limit = 5): GitCommit[] {
  const output = exec(
    root,
    `git log --follow -n ${limit} --format='%H|%an|%ae|%at|%s' -- ${filePath}`,
  );
  if (!output) return [];
  return output
    .split("\n")
    .filter(Boolean)
    .map((line) => {
      const [hash, author, email, date, ...msgParts] = line.split("|");
      return { hash, author, email, date: parseInt(date, 10), message: msgParts.join("|") };
    });
}
```

### 1.4 `enrichErrorStack()` 函数

**新增文件：** `src/enrich.ts`

**新增类型（`src/types.ts`）：**

```typescript
interface ErrorFrame {
  file: string;
  line: number;
  column?: number;
  function?: string;
}

interface EnrichedFrame {
  file: string;
  line: number;
  column?: number;
  function?: string;
  // kly index
  fileDescription: string;
  fileSummary: string;
  symbols: SymbolInfo[];
  language: Language;
  // kly dependencies
  importedBy: string[];
  importsFrom: string[];
  // git
  lastModified: GitCommit | null;
  recentCommits: GitCommit[];
}

interface EnrichedErrorStack {
  frames: EnrichedFrame[];
  affectedFiles: number; // 所有 frame 的 importedBy 去重总数
}
```

**实现逻辑：**

```
对每个 frame:
  1. db.getFile(frame.file)         → description, summary, symbols, language
  2. db.getDependents(frame.file)   → importedBy（走 dependencies 表，O(1)）
  3. db.getDependencies(frame.file) → importsFrom（走 dependencies 表，O(1)）
  4. getFileHistory(root, frame.file, 5) → lastModified + recentCommits
  5. 组装 EnrichedFrame

最后计算 affectedFiles = 所有 frame 的 importedBy 去重
```

**注意：** 如果文件不在 index 中，该 frame 返回空值填充（不报错，降级处理）。

## Phase 2：CLI 全面改造

### 2.1 输出基础设施

**新增文件：** `src/commands/output.ts`

统一的输出工具，所有命令共用：

```typescript
interface OutputOptions {
  pretty?: boolean;
}

/** 输出数据。默认 JSON，--pretty 时人类可读 */
function output(data: unknown, opts: OutputOptions): void;

/** 输出错误到 stderr，附带正确用法 */
function error(message: string, hint?: string): never;

/** 输出警告到 stderr */
function warn(message: string): void;
```

- `output()` 默认 `JSON.stringify(data)`，`--pretty` 时调用各命令自定义的 formatter
- `error()` 输出到 stderr 并 `process.exit(1)`
- 所有命令不再 import `@clack/prompts`

---

### 2.2 `kly init` — 非交互式改造

**改动文件：** `src/commands/init.ts`、`src/cli.ts`

新增 flags：

```
--provider <name>    LLM provider
--model <name>       Model name
--api-key <key>      API key
--hook               Install post-commit hook
--include <glob>     Include patterns (repeatable)
--exclude <glob>     Exclude patterns (repeatable)
```

逻辑：

- 如果 `--provider` + `--api-key` 都给了 → 非交互式，直接写 config
- 如果缺 flag → fallback 到交互式（但交互式也要从 clack 换成更简单的 readline，或者保留 clack 只在这一个命令里——因为 init 是唯一需要交互的命令）

**决策：** 保留 clack 做交互 fallback。init 是唯一交互命令，且只在人类初始化时用。

### 2.3 现有命令改造

每个命令都要：

1. 去掉 `@clack/prompts`
2. 默认输出 JSON
3. 加 `--pretty` flag
4. `--help` 加 examples
5. 错误输出到 stderr

**`src/commands/query.ts`**

```
$ kly query "auth"
→ JSON: [{ path, name, description, score, symbols }]

$ kly query "auth" --pretty
→ found 3 file(s)
  src/auth.ts
    name: Authentication Service
    description: Handles JWT validation
    score: 12.35
```

**`src/commands/show.ts`**

```
$ kly show src/auth.ts
→ JSON: { path, name, description, language, imports, exports, symbols, summary, hash, indexedAt }

$ kly show src/auth.ts --pretty
→ path: src/auth.ts
  name: Authentication Service
  language: typescript
  ...
```

**`src/commands/overview.ts`**

```
$ kly overview
→ JSON: { totalFiles, languages, files: [{ path, name, description }] }

$ kly overview --pretty
→ total_files: 42
  languages:
    typescript: 35 (83.3%)
    swift: 7 (16.7%)
  ...
```

**`src/commands/graph.ts`**

去掉 `--format ascii|mermaid|svg`，统一为：

```
$ kly graph
→ JSON: { nodes: [...], edges: [...] }

$ kly graph --pretty
→ 简化 ASCII 列表（人类可读的依赖关系树形展示）
  例如：
  src/auth.ts
    → src/types.ts
    → src/database.ts
    ← src/middleware/guard.ts
    ← src/routes/login.ts

$ kly graph --focus src/auth.ts --depth 3
```

**决策：** `--pretty` 输出简化 ASCII 列表，不是 mermaid。mermaid 对人不可读。

去掉 `beautiful-mermaid` 依赖（ASCII 渲染和 SVG 渲染都不需要了）。
`generateMermaid()` 函数保留在库中（Santi 或其他消费者可能用），但 CLI 不再使用。

**`src/commands/build.ts`**

build 是写入操作，不需要 `--json`/`--pretty`。但成功输出改为结构化：

```
$ kly build
→ indexed 12 files (3 new, 9 unchanged)
  branch: main
  commit: abc1234
  duration: 2.3s

$ kly build --quiet
→ （无输出，给 git hook 用）
```

**决策：** 改 `buildIndex()` 返回 `BuildResult`（当前返回 void）。改动面大但信息有用。

```typescript
interface BuildResult {
  totalFiles: number;
  newFiles: number;
  updatedFiles: number;
  deletedFiles: number;
  unchangedFiles: number;
  branch: string;
  commit: string;
  duration: number;
}
```

需要改动 `indexer.ts` 的 `buildIndex()`、`buildGitAware()`、`buildClassic()`、`buildFromGitDiff()` 全链路返回值。

**`src/commands/shared.ts`**

`ensureInitialized` 去掉 clack，改用 stderr 输出：

```typescript
export function ensureInitialized(root: string): void {
  if (!isInitialized(root)) {
    process.stderr.write(
      "Error: Not initialized.\n  kly init --provider <provider> --api-key <key>\n",
    );
    process.exit(1);
  }
}
```
